Dies ist die zweite Aufgabe zum Parsen von YAML. Sie stellt den nächsten Schritt dar, bei dem das Parsen komplizierter wird. Die Datentypen, wie z.B {% if is_js %}null{% else %}None{% endif %} und {% if is_js %}boolean{% else %}bool{% endif %}werden hinzugefügt, und außerdem erhalten Sie die Möglichkeit, Anführungszeichen in Zeichenketten zu verwenden.

Hier sind einige der Beispiele:

name: "Bob Dylan"
children: 6

{
  "name": "Bob Dylan", 
  "children": 6
}      
      
name: "Alex \\"Fox\\""
children: 6

{
  "name": "Alex \"Fox\"",
  "children": 6
}

Wie Sie sehen können, kann die Zeichenkette in Anführungszeichen gesetzt werden. Es können sowohl doppelte als auch einfache Anführungszeichen sein.

name: "Bob Dylan"
children: 6
alive: false
        
{
  "name": "Bob Dylan", 
  "alive": {% if is_js %}false{% else %}False{% endif %}, 
  "children": 6
}

name: "Bob Dylan"
children: 6

{
  "coding": {% if is_js %}null{% else %}None{% endif %}, 
  "name": "Bob Dylan", 
  "children": 6
}

Wenn kein Wert angegeben wird, wird er undefiniert. Auch hierfür gibt es ein Schlüsselwort - {% if is_js %}null{% else %}None{%endif%}.
