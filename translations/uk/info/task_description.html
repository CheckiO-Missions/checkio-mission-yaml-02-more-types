Це друга місія щодо парсингу YAML. Вона являє собою наступний крок, де парсинг ускладнюється. Вводяться такі типи даних, як {% if is_js %}null{% else %}None{% endif %} і логічний (булевий), і крім цього, в рядках починають використовуватися лапки.

Ось деякі з прикладів:

name: "Bob Dylan"
children: 6

{
  "name": "Bob Dylan", 
  "children": 6
}      
      
name: "Alex \\"Fox\\""
children: 6

{
  "name": "Alex \"Fox\"",
  "children": 6
}

Як бачиш, рядок можна поставити в лапки. Це можуть бути як подвійні, так і одиночні лапки.

name: "Bob Dylan"
children: 6
alive: false
        
{
  "name": "Bob Dylan", 
  "alive": {% if is_js %}false{% else %}False{% endif %}, 
  "children": 6
}

{% if is_js %}t{% else %}T{% endif %}rue і {% if is_js %}f{% else %}F{% endif %}alse - це значення логічного (булевого) типу.

name: "Bob Dylan"
children: 6
coding:
        
{
  "coding": {% if is_js %}null{% else %}None{% endif %}, 
  "name": "Bob Dylan", 
  "children": 6
}

Якщо ніяке значення не вказане, воно стає невизначеним. Для цього також є ключове слово - {% if is_js %}null{% else %}None{%endif%}.

Вхідні дані: Форматований рядок.

Вихідні дані: об'єкт ({% if is_js %}масив/словник{% else %}список/словник{%endif%}).

Приклади: 

{{init_code_tmpl}}

Передумова: YAML 1.2 використовується з JSON Schema.
