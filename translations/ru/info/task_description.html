Это вторая задача по парсингу YAML. Она представляет собой следующий этап усложнения парсинга, в котором добавляются типы данных, такие как {% if is_js %}null{% else %}None{% endif %} и логический (булевый), а также возможность использовать кавычки в строках.

Вот несколько примеров:

name: "Bob Dylan"
children: 6

{
  "name": "Bob Dylan", 
  "children": 6
}      
      
name: "Alex \\"Fox\\""
children: 6

{
  "name": "Alex \"Fox\"",
  "children": 6
}

Как видите, строку можно задавать в кавычках. Это могут быть как двойные, так и одинарные кавычки.

name: "Bob Dylan"
children: 6
alive: false
        
{
  "name": "Bob Dylan", 
  "alive": {% if is_js %}false{% else %}False{% endif %}, 
  "children": 6
}

{% if is_js %}t{% else %}T{% endif %}rue и {% if is_js %}f{% else %}F{% endif %}alse - это ключевые слова, определяющие булевный тип.

name: "Bob Dylan"
children: 6
coding:
        
{
  "coding": {% if is_js %}null{% else %}None{% endif %}, 
  "name": "Bob Dylan", 
  "children": 6
}

Если значение не указано, то оно становится неопределенным. Также для этого есть ключевое слово {% if is_js %}null{% else %}None{%endif%}.

Входные данные: Строка в формате

Выходные данные: Объект ({% if is_js %}массив/словарь{% else %}список/словарь{%endif%}).

Примеры: 

{{init_code_tmpl}}

Предварительное условие: Используется стандарт YAML версии 1.2
